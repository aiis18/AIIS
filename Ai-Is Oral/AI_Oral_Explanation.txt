
ORAL EXPLANATION FOR AI PRACTICAL PROGRAMS

====================================================
1. A* SEARCH ALGORITHM
====================================================

What is A* Algorithm?
A* (A-Star) is an informed search algorithm used to find the shortest path between two nodes in a graph.

Formula:
f(n) = g(n) + h(n)

Where:
g(n) = actual cost from start node
h(n) = heuristic estimated cost
f(n) = total estimated cost

Modules Used:
import heapq

heapq is used to implement Priority Queue.

Main Concepts:
- Open List stores nodes to explore
- Parent dictionary stores shortest path
- g_cost stores path cost
- Priority Queue selects minimum cost node first

Functions:
1. a_star()
   Main search function.

2. reconstruct_path()
   Reconstructs final shortest path.

Advantages:
- Fast
- Optimal
- Finds shortest path

Applications:
- GPS Navigation
- Robotics
- Games

Time Complexity:
O(E log V)

====================================================
2. GRAPH COLORING USING BACKTRACKING & BRANCH AND BOUND
====================================================

What is Graph Coloring?
Assign colors to graph vertices such that adjacent vertices do not have same color.

Functions:
1. is_safe()
   Checks whether color assignment is valid.

2. graph_coloring_bt()
   Uses Backtracking approach.

3. graph_coloring_bb()
   Uses Branch and Bound approach.

Backtracking:
- Tries all possibilities
- Goes back if solution fails

Branch and Bound:
- Optimized version
- Prunes invalid branches early

Time Complexity:
O(m^n)

Applications:
- Map Coloring
- Timetable Scheduling
- Register Allocation

====================================================
4. DFS AND BFS TRAVERSAL
====================================================

Graph Traversal:
Process of visiting all nodes in graph.

DFS (Depth First Search):
- Explores depth wise
- Uses recursion/stack

Function:
dfs()

BFS (Breadth First Search):
- Explores level wise
- Uses Queue

Function:
bfs()

Important Concepts:
- visited set prevents revisiting
- queue used in BFS
- recursion used in DFS

Difference Between DFS and BFS:

DFS:
- Depth Wise
- Uses Stack
- Less memory

BFS:
- Level Wise
- Uses Queue
- Finds shortest path

Time Complexity:
O(V + E)

Applications:
- Web Crawling
- Social Networks
- Network Broadcasting

====================================================
5. GREEDY BEST FIRST SEARCH
====================================================

What is Greedy Best First Search?
Search algorithm that selects node with smallest heuristic value.

Formula:
f(n) = h(n)

Unlike A*:
- Does not use path cost g(n)

Modules Used:
import heapq

Main Components:
- Open List
- Visited Set
- Parent Dictionary

Functions:
1. greedy_best_first_search()
   Main search function.

2. reconstruct_path()
   Builds final path.

Advantages:
- Faster
- Simple implementation

Disadvantages:
- May not give shortest path

Applications:
- AI Navigation
- Robot Movement
- Game Pathfinding

Time Complexity:
O(E log V)

====================================================
COMMON VIVA QUESTIONS
====================================================

1. What is heuristic function?
Heuristic estimates cost from current node to goal.

2. Why use Priority Queue?
To always select minimum cost node first.

3. What is Backtracking?
Trying solutions recursively and undoing wrong choices.

4. Difference between BFS and DFS?
BFS is level-wise traversal while DFS is depth-wise traversal.

5. What is adjacency matrix?
Matrix representation of graph connections.

6. Why visited set is used?
To avoid revisiting nodes.

7. Which algorithm guarantees shortest path?
- BFS in unweighted graph
- A* with admissible heuristic

8. What is recursion?
Function calling itself repeatedly.
